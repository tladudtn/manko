
     -M, --machine=
         로컬 컨테이너에서 작업을 실행합니다. 연결할 컨테이너 이름을 지정하며, 연결할 사용자 이름을 접두사로 사용하고 구분 기호 "@"를 사용할 수 있습니다. 컨테이너 이름 대신 특수 문자 ".host"를 사용하면 로컬 시스템에 연결됩니다 (특정 사용자의 사용자 버스에 연결하는 데 유용합니다: "--user --machine=lennart@.host"). "@" 구문을 사용하지 않으면 root 사용자로 연결됩니다. "@" 구문을 사용하면 왼쪽 또는 오른쪽 측면을 생략할 수 있지만 (둘 다는 안 됩니다), 이 경우 로컬 사용자 이름과 ".host"가 암시됩니다.

     -C, --capsule=
         캡슐에서 작업을 실행합니다. 연결할 캡슐 이름을 지정합니다. 캡슐에 대한 자세한 내용은 capsule@.service(5)를 참조하십시오.

         버전 256에서 추가되었습니다.

     --no-ask-password
         특권 작업에 대해 사용자에게 인증을 요청하지 않습니다.

     --no-pager
         출력을 페이저로 파이프하지 않습니다.

     --legend=BOOL
         범례, 즉 열 헤더와 힌트가 포함된 바닥글의 인쇄를 활성화하거나 비활성화합니다. 범례는 기본적으로 인쇄되지만, --quiet 또는 유사한 옵션으로 비활성화할 수 있습니다.

     -h, --help
         간단한 도움말 텍스트를 출력하고 종료합니다.

     --version
         간단한 버전 문자열을 출력하고 종료합니다.

EXIT STATUS
     성공 시 0이 반환되며, 그 외에는 0이 아닌 실패 코드가 반환됩니다.

     systemctl은 LSB 3.0.0[3]에 정의된 LSB가 정의한 반환 코드를 사용합니다.

     Table 5. LSB 반환 코드
     ┌───────┬──────────────────────┬──────────────────────┐
     │ Value │ Description in LSB   │ Use in systemd       │
     ├───────┼──────────────────────┼──────────────────────┤
     │ 0     │ "program is  running │ unit is active       │
     │       │ or service is OK"    │                      │
     ├───────┼──────────────────────┼──────────────────────┤
     │ 1     │ "program is dead and │ unit   not    failed │
     │       │ /var/run   pid  file │ (used by is-failed)  │
     │       │ exists"              │                      │
     ├───────┼──────────────────────┼──────────────────────┤
     │ 2     │ "program is dead and │ unused               │
     │       │ /var/lock  lock file │                      │
     │       │ exists"              │                      │
     ├───────┼──────────────────────┼──────────────────────┤
     │ 3     │ "program is not run‐ │ unit is not active   │
     │       │ ning"                │                      │
     ├───────┼──────────────────────┼──────────────────────┤
     │ 4     │ "program or  service │ no such unit         │
     │       │ status is unknown"   │                      │
     └───────┴──────────────────────┴──────────────────────┘

     LSB 서비스 상태와 systemd 유닛 상태 간의 매핑은 완벽하지 않으므로, 해당 반환 값에 의존하기보다는 특정 유닛 상태 및 하위 상태를 찾는 것이 더 좋습니다.

ENVIRONMENT
     $SYSTEMD_EDITOR
         유닛을 편집할 때 사용할 편집기입니다. $EDITOR 및 $VISUAL을 재정의합니다. $SYSTEMD_EDITOR, $EDITOR, $VISUAL 중 어느 것도 없거나 빈 문자열로 설정되었거나 실행에 실패한 경우, systemctl은 다음 순서로 잘 알려진 편집기를 실행하려고 시도합니다: editor(1), nano(1), vim(1), vi(1).

         버전 218에서 추가되었습니다.

     $SYSTEMD_LOG_LEVEL
         방출되는 메시지의 최대 로그 레벨입니다 (더 높은 로그 레벨, 즉 덜 중요한 메시지는 억제됩니다). 쉼표로 구분된 값 목록을 받습니다. 값은 (중요도 감소 순서로) emerg, alert, crit, err, warning, notice, info, debug 중 하나이거나 0...7 범위의 정수일 수 있습니다. 자세한 내용은 syslog(3)를 참조하십시오. 각 값은 해당 특정 로그 대상을 위한 최대 로그 레벨을 설정하기 위해 console, syslog, kmsg 또는 journal 중 하나로 접두사 지정된 후 콜론을 붙일 수 있습니다 (예: SYSTEMD_LOG_LEVEL=debug,console:info는 콘솔에 로깅하는 경우 info 레벨이어야 하는 것을 제외하고 debug 레벨로 로깅하도록 지정합니다). 전역 최대 로그 레벨이 대상별 최대 로그 레벨보다 우선한다는 점에 유의하십시오.

     $SYSTEMD_LOG_COLOR
         불리언 값입니다. true인 경우, tty에 작성된 메시지는 우선순위에 따라 색상이 지정됩니다.

         이 설정은 메시지가 터미널에 직접 작성될 때만 유용합니다. journalctl(1) 및 로그를 표시하는 다른 도구는 자체적으로 로그 레벨에 따라 메시지에 색상을 지정하기 때문입니다.

     $SYSTEMD_LOG_TIME
         불리언 값입니다. true인 경우, 콘솔 로그 메시지에 타임스탬프가 접두사로 지정됩니다.

         이 설정은 메시지가 터미널이나 파일에 직접 작성될 때만 유용합니다. journalctl(1) 및 로그를 표시하는 다른 도구는 자체적으로 항목 메타데이터를 기반으로 타임스탬프를 첨부하기 때문입니다.

     $SYSTEMD_LOG_LOCATION
         불리언 값입니다. true인 경우, 메시지는 메시지가 발생하는 소스 코드의 파일 이름과 줄 번호로 접두사 지정됩니다.

         로그 위치는 어쨌든 journal 항목에 메타데이터로 첨부되는 경우가 많다는 점에 유의하십시오. 이를 메시지 텍스트에 직접 포함하는 것은 프로그램 디버깅 시 편리할 수 있습니다.

     $SYSTEMD_LOG_TARGET
         로그 메시지의 목적지입니다. console (첨부된 tty에 로깅), console-prefixed (첨부된 tty에 로깅하지만 로그 레벨과 "facility"를 인코딩하는 접두사를 사용, syslog(3) 참조), kmsg (커널 순환 로그 버퍼에 로깅), journal (저널에 로깅), journal-or-kmsg (사용 가능한 경우 저널에, 그렇지 않으면 kmsg에 로깅), auto (적절한 로그 대상을 자동으로 결정, 기본값), null (로그 출력 비활성화) 중 하나입니다.

     $SYSTEMD_PAGER, $PAGER
         --no-pager가 지정되지 않은 경우 사용할 페이저입니다. $SYSTEMD_PAGER가 설정되어 있으면 그것이 사용되며, 그렇지 않으면 $PAGER가 사용됩니다. $SYSTEMD_PAGER와 $PAGER 둘 다 설정되지 않은 경우, less(1) 및 more(1)을 포함하여 잘 알려진 페이저 구현 세트를 차례로 시도하여 하나를 찾습니다. 페이저 구현을 찾지 못하면 페이저가 호출되지 않습니다. 해당 환경 변수를 빈 문자열 또는 "cat" 값으로 설정하는 것은 --no-pager를 전달하는 것과 동일합니다.

         참고: $SYSTEMD_PAGERSECURE가 설정되어 있지 않으면, $SYSTEMD_PAGER와 $PAGER는 페이저를 비활성화하는 데 ( "cat" 또는 ""로)만 사용될 수 있으며, 그 외에는 무시됩니다.

     $SYSTEMD_LESS
         less에 전달되는 옵션을 재정의합니다 (기본값은 "FRSXMK").

         사용자는 특히 두 가지 옵션을 변경하고 싶을 수 있습니다:

         K
             이 옵션은 Ctrl+C가 눌렸을 때 페이저가 즉시 종료하도록 지시합니다. less가 자체적으로 Ctrl+C를 처리하여 페이저 명령 프롬프트로 다시 전환할 수 있도록 하려면 이 옵션을 해제하십시오.

             $SYSTEMD_LESS의 값이 "K"를 포함하지 않고 호출된 페이저가 less인 경우, Ctrl+C는 실행 파일에 의해 무시되며 페이저에 의해 처리되어야 합니다.

         X
             이 옵션은 페이저가 termcap 초기화 및 비초기화 문자열을 터미널로 전송하지 않도록 지시합니다. 이는 페이저가 종료된 후에도 명령 출력이 터미널에 계속 표시되도록 기본적으로 설정됩니다. 그럼에도 불구하고, 이는 일부 페이저 기능이 작동하지 않도록 방지하며, 특히 마우스로 페이지 출력을 스크롤할 수 없습니다.

         일반 $LESS 환경 변수를 설정하는 것은 systemd 도구에 의한 less 호출에는 영향을 미치지 않는다는 점에 유의하십시오.

         더 자세한 논의는 less(1)를 참조하십시오.
