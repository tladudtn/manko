
ENVIRONMENT

     $SYSTEMD_LESSCHARSET
         less에 전달되는 문자 집합을 재정의합니다 (기본값은 "utf-8"이며, 호출 터미널이 UTF-8 호환으로 결정된 경우).

         일반적인 $LESSCHARSET 환경 변수를 설정하는 것은 systemd 도구에 의한 less 호출에는 영향을 미치지 않는다는 점에 유의하십시오.

     $SYSTEMD_PAGERSECURE
         less(1)와 같은 일반적인 페이지 매김 명령어는 "페이지 매김", 즉 출력 스크롤링 외에도 다른 파일 열기 또는 쓰기 및 임의의 셸 명령 실행을 지원합니다. 명령이 sudo(8) 또는 pkexec(1)과 같은 높은 권한으로 호출될 때, 페이지 매김기는 보안 경계가 됩니다. 엄격하게 제한된 기능만을 가진 프로그램이 페이지 매김기로 사용되도록 주의해야 하며, 새 파일 열기나 생성 또는 서브프로세스 시작과 같은 의도하지 않은 대화형 기능은 허용되지 않습니다. 페이지 매김기가 이를 지원하는 경우, 아래에 설명된 대로 페이지 매김기의 "보안 모드"를 활성화할 수 있습니다 (대부분의 페이지 매김기는 이를 고려하여 작성되지 않았습니다). 신뢰할 수 없는 사용자가 높은 권한으로 명령을 실행하도록 허용할 때는 "보안 모드"를 명시적으로 활성화하거나 --no-pager 또는 PAGER=cat을 사용하여 페이지 매김기를 완전히 비활성화하는 것이 좋습니다.

         이 옵션은 부울 인수를 받습니다. true로 설정되면 페이지 매김기의 "보안 모드"가 활성화됩니다. "보안 모드"에서는 페이지 매김기를 호출할 때 LESSSECURE=1이 설정되며, 이는 페이지 매김기에게 새 파일을 열거나 생성하거나 새 서브프로세스를 시작하는 명령을 비활성화하도록 지시합니다. 현재 이 변수를 이해하고 "보안 모드"를 구현하는 것으로 알려진 것은 less(1)뿐입니다.

         false로 설정되면 페이지 매김기에 제한이 가해지지 않습니다. SYSTEMD_PAGERSECURE=0을 설정하거나 상속된 환경에서 이를 제거하지 않으면 사용자가 임의의 명령을 호출할 수 있게 될 수 있습니다.

         $SYSTEMD_PAGERSECURE가 설정되지 않은 경우, systemd 도구는 "보안 모드"가 활성화되어야 하는지 그리고 페이지 매김기가 이를 지원하는지 자동으로 파악하려고 시도합니다. 효과적인 UID가 로그인 세션 소유자와 같지 않은 경우 (geteuid(2) 및 sd_pid_get_owner_uid(3) 참조) 또는 sudo(8) 또는 유사한 도구($SUDO_UID가 설정된 경우 [4])에서 실행되는 경우 "보안 모드"가 활성화됩니다. 그러한 경우, SYSTEMD_PAGERSECURE=1이 설정되며 "보안 모드"를 구현하는 것으로 알려지지 않은 페이지 매김기는 전혀 사용되지 않습니다. 이 자동 감지는 권한을 상승시키는 가장 일반적인 메커니즘만을 다루며 편의를 위한 것임을 유의하십시오. $SYSTEMD_PAGERSECURE를 명시적으로 설정하거나 페이지 매김기를 비활성화하는 것이 권장됩니다.

         $SYSTEMD_PAGER 또는 $PAGER 변수가 페이지 매김기를 비활성화하는 것 외에 존중되려면, $SYSTEMD_PAGERSECURE도 설정되어야 합니다.

     $SYSTEMD_COLORS
         부울 인수 또는 특수 값을 받습니다. 기본값(설정되지 않음)으로, systemd 및 관련 유틸리티는 가능한 경우 출력에 색상을 사용합니다. $COLORTERM이 "truecolor" 또는 "24bit"로 설정된 경우 24비트 색상이 활성화되며, 그렇지 않은 경우 256색이 활성화됩니다. 단, $NO_COLOR 또는 $TERM이 색상이 비활성화되었음을 나타내는 경우는 제외합니다.

         true
             설정되지 않은 경우와 동일하지만 $NO_COLOR는 무시됩니다.

         false
             출력은 단색이 됩니다.

         "16", "256", "24bit"
             각각 기본 16 ANSI 색상, 256색 또는 24비트 색상을 항상 사용합니다.

         "auto-16", "auto-256", "auto-24bit"
             $TERM 및 콘솔이 연결된 환경에 따라 주어진 수의 색상을 사용합니다.

     $SYSTEMD_URLIFY
         값은 부울이어야 합니다. 이 기능을 지원하는 터미널 에뮬레이터의 출력에 클릭 가능한 링크가 생성될지 여부를 제어합니다. 이는 $TERM 및 기타 조건에 따라 systemd가 내리는 결정을 재정의하기 위해 지정될 수 있습니다.
