
   Environment Commands
     systemd는 매니저가 생성하는 프로세스에 전달되는 환경 블록을 지원합니다. 변수 이름에는 ASCII 문자, 숫자, 밑줄 문자를 포함할 수 있습니다. 변수 이름은 비어 있거나 숫자로 시작할 수 없습니다. 변수 값에서는 대부분의 문자가 허용되지만, 전체 시퀀스는 유효한 UTF-8이어야 합니다. (줄 바꿈(NL), 탭(TAB), 또는 이스케이프 문자(ESC)와 같은 제어 문자는 유효한 ASCII이며 따라서 유효한 UTF-8이라는 점에 유의하십시오). 환경 블록의 전체 길이는 sysconf(3)에 정의된 _SC_ARG_MAX 값으로 제한됩니다.

     show-environment
         systemd 매니저 환경 블록을 출력합니다. 이는 매니저가 생성하는 모든 프로세스에 전달되는 환경 블록입니다. 환경 블록은 대부분의 셸에서 소싱하기에 적합한 단순한 형태로 출력됩니다. 변수 값에 특수 문자나 공백이 없는 경우 이스케이프 처리는 수행되지 않으며 할당은 "VARIABLE=value" 형태를 가집니다. 공백이나 셸에서 특별한 의미를 갖는 문자가 있는 경우 달러-단일-따옴표 이스케이프가 사용되며 할당은 "VARIABLE=$'value'" 형태를 가집니다. 이 구문은 bash(1), zsh(1), ksh(1), 그리고 busybox(1)의 ash(1)에서 지원되는 것으로 알려져 있지만 dash(1)나 fish(1)에서는 지원되지 않습니다.

         이것은 유효한 블록, 즉 구성 파일, 환경 생성기, 그리고 IPC(아래에 설명된 set-environment를 통해)를 통해 구성된 환경 변수의 조합을 보여준다는 점에 유의하십시오. 유닛 프로세스가 포크될 때, 이 결합된 환경 블록은 단위별 환경 변수와 추가로 결합되며, 이 변수들은 이 명령어에서는 보이지 않습니다.

     set-environment VARIABLE=VALUE...
         명령줄에 지정된 하나 이상의 서비스 매니저 환경 변수를 설정합니다. 이 명령어는 변수 이름과 값이 위에 나열된 규칙을 준수하지 않으면 실패합니다.

         이것이 서비스 매니저 구성 및 환경 생성기로부터 구성된 환경 블록과는 별개의 환경 블록에 작동한다는 점에 유의하십시오. 프로세스가 호출될 때마다 두 블록은 결합되고 (서비스별 환경 변수도 통합되며) 프로세스에 전달됩니다. show-environment 명령은 블록의 조합을 보여줍니다. 위를 참조하십시오.

         버전 233에 추가되었습니다.

     unset-environment VARIABLE...
         하나 이상의 systemd 매니저 환경 변수를 해제합니다. 변수 이름만 지정된 경우, 값에 관계없이 제거됩니다. 변수와 값이 지정된 경우, 해당 변수는 지정된 값과 일치할 때만 제거됩니다.

         이것이 서비스 매니저 구성 및 환경 생성기로부터 구성된 환경 블록과는 별개의 환경 블록에 작동한다는 점에 유의하십시오. 프로세스가 호출될 때마다 두 블록은 결합되고 (서비스별 환경 변수도 통합되며) 프로세스에 전달됩니다. show-environment 명령은 블록의 조합을 보여줍니다. 위를 참조하십시오. 이것은 이 명령을 서비스 매니저 구성 파일이나 생성기를 통해 정의된 환경 변수를 해제하는 데 사용할 수 없다는 것을 의미합니다.

         버전 233에 추가되었습니다.

     import-environment VARIABLE...
         클라이언트에서 설정된 모든 하나 이상의 환경 변수를 systemd 매니저 환경 블록으로 가져옵니다. 환경 변수 이름 목록이 전달되면, 클라이언트 측 값은 매니저의 환경 블록으로 가져와집니다. 위에 설명된 규칙에 따라 유효한 환경 변수 이름이 아니거나 유효하지 않은 값을 가진 이름이 있으면 오류가 발생합니다. 인수가 전달되지 않으면 systemctl 프로세스가 상속받은 전체 환경 블록이 가져와집니다. 이 모드에서는 상속된 유효하지 않은 환경 변수는 조용히 무시됩니다.

         전체 상속된 환경 블록을 가져오는 것(인수 없이 이 명령을 호출하는 것)은 사용 중단되었습니다. 셸은 로컬에서만 의미가 있고 셸의 자손 프로세스만을 위한 수십 개의 변수를 설정합니다. 이러한 변수들이 전역 환경 블록에 있으면 다른 프로세스에게 혼란을 줍니다.

         버전 209에 추가되었습니다.

   Manager State Commands
     daemon-reload
         systemd 매니저 구성을 다시 로드합니다. 이는 모든 생성기(systemd.generator(7) 참조)를 다시 실행하고, 모든 유닛 파일을 다시 로드하며, 전체 종속성 트리를 재구성합니다. 데몬이 다시 로드되는 동안, systemd가 사용자 구성을 대신하여 수신하는 모든 소켓은 계속 액세스 가능합니다.

         이 명령어는 reload 명령어와 혼동해서는 안 됩니다.

     daemon-reexec
         systemd 매니저를 재실행합니다. 이는 매니저 상태를 직렬화하고, 프로세스를 재실행한 다음, 상태를 다시 역직렬화합니다. 이 명령어는 디버깅 및 패키지 업그레이드를 제외하고는 거의 사용되지 않습니다. 때로는 무거운 daemon-reload로 유용할 수 있습니다. 데몬이 재실행되는 동안, systemd가 사용자 구성을 대신하여 수신하는 모든 소켓은 계속 액세스 가능합니다.

     log-level [LEVEL]
         인수가 제공되지 않으면 매니저의 현재 로그 레벨을 출력합니다. 선택적 인수 LEVEL이 제공되면, 이 명령어는 매니저의 현재 로그 레벨을 LEVEL로 변경합니다 (systemd(1)에 설명된 --log-level=과 동일한 값을 허용합니다).

         버전 244에 추가되었습니다.

     log-target [TARGET]
         인수가 제공되지 않으면 매니저의 현재 로그 대상을 출력합니다. 선택적 인수 TARGET이 제공되면, 이 명령어는 매니저의 현재 로그 대상을 TARGET로 변경합니다 (systemd(1)에 설명된 --log-target=과 동일한 값을 허용합니다).

         버전 244에 추가되었습니다.

     service-watchdogs [yes|no]
         인수가 제공되지 않으면 매니저의 서비스 런타임 워치독의 현재 상태를 출력합니다. 선택적 부울 인수가 제공되면, 서비스 런타임 워치독(WatchdogSec=)과 비상 조치(예: OnFailure= 또는 StartLimitAction=)를 전역적으로 활성화하거나 비활성화합니다. systemd.service(5)를 참조하십시오. 하드웨어 워치독은 이 설정의 영향을 받지 않습니다.

         버전 244에 추가되었습니다.
